# DocPress — Serverless Publishing Platform

> **Neo-brutalist • Static • Zero-maintenance** publishing powered by Google Drive, Next.js SSG, and Firebase Hosting.

---

## Architecture

```
Google Drive (.docx) ──► GitHub Actions ──► Next.js Build ──► Firebase Hosting (CDN)
                                │               │
                         mammoth.js parse    Tailwind CSS
                         Buffer streaming    Typography plugin
```

---

## Quick Start

### Prerequisites
- Node.js 20+
- A Google Cloud project with Drive API enabled
- A Firebase project with Hosting enabled
- A GitHub repository

---

## Phase 1 — Google Cloud Setup

### 1. Create Service Account
1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Enable the **Google Drive API** in your project
3. Navigate to **IAM & Admin → Service Accounts → Create**
4. Give it a name (e.g., `docpress-builder`)
5. Generate a **JSON key** and download it securely

### 2. Share your Drive folder
1. Create a folder in Google Drive named `Published Docs` (or any name)
2. Copy the **Folder ID** from the URL: `drive.google.com/drive/folders/FOLDER_ID`
3. Right-click the folder → Share → paste the Service Account email → set **Viewer**

---

## Phase 2 — Local Setup

```bash
cd publishing-platform
cp .env.local.example .env.local
```

Edit `.env.local`:
```
GOOGLE_SERVICE_ACCOUNT_EMAIL=your-sa@your-project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\n...\n-----END RSA PRIVATE KEY-----"
GOOGLE_DRIVE_FOLDER_ID=your_folder_id_here
```

> **Tip:** In the JSON key file, the `private_key` field already contains `\n` escape sequences — paste it exactly as-is, wrapped in double quotes.

```bash
npm install
npm run dev     # development server
npm run build   # static export → out/
```

---

## Phase 3 — Firebase Hosting

```bash
npm install -g firebase-tools
firebase login
firebase init hosting
# → select "Use an existing project"
# → public dir: out
# → single page app: NO
# → overwrite index.html: NO
```

Test locally:
```bash
npm run build && firebase serve --only hosting
```

## Phase 4 — CI/CD (GitHub Actions)

### Encode Service Account for GitHub Secrets

**Linux/macOS:**
```bash
base64 -w 0 your-service-account.json
```
**Windows (PowerShell):**
```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("your-service-account.json"))
```

### Add GitHub Secrets

| Secret Name | Value |
|---|---|
| `GOOGLE_CREDENTIALS_BASE64` | Base64-encoded SA JSON |
| `GOOGLE_SERVICE_ACCOUNT_EMAIL` | SA email address |
| `GOOGLE_PRIVATE_KEY` | Private key from SA JSON |
| `GOOGLE_DRIVE_FOLDER_ID` | Drive folder ID |
| `FIREBASE_SERVICE_ACCOUNT` | Generated by `firebase init hosting:github` |
| `FIREBASE_PROJECT_ID` | Your Firebase project ID |

The workflow at `.github/workflows/deploy.yml` fires on:
- Manual dispatch (`workflow_dispatch`)
- `repository_dispatch` with `event_type: drive-update`

---

## Phase 5 — Auto-trigger (Google Apps Script)

1. Open [script.google.com](https://script.google.com) → New Project
2. Paste the contents of `scripts/drive-trigger.gs`
3. Go to **Project Settings → Script Properties** → add:
   - `GH_TOKEN` — GitHub PAT with `repo` scope
   - `GH_REPO` — `username/repository`
   - `DRIVE_FOLDER_ID` — same folder ID as above
4. Run `initializeState()` once manually
5. Add trigger: **Triggers → + Add → `checkDriveForChanges` → Time-based → Every 10 minutes**

---

## Adding Documents

1. Write your document in Microsoft Word or Google Docs
2. Export as `.docx`
3. Drop the `.docx` file into your configured Google Drive folder
4. Wait ~10 minutes for the Apps Script to detect the change and trigger the build
5. Done — your document is live ✦
